# 📋 Linker(링커) — 서비스 기획서 (PRD)

> **아동보호전문기관 사회복지사를 위한 국가아동학대정보시스템 DB 자동 입력 솔루션**

---

## 1. 서비스 개요

### 1.1 한 줄 정의
> 엑셀에 정리해둔 상담·서비스 기록을 드래그 앤 드롭 하나로 **국가아동학대정보시스템**에 자동 입력해주는 **Windows 데스크톱 RPA 프로그램**

### 1.2 배경 및 문제 인식
전국 아동보호전문기관의 사회복지사(상담원)들은 아동학대 사례에 대한 상담·서비스를 수행한 후, 다음과 같은 **이중 기록** 업무를 수행하고 있다:

1. **1차 기록**: 상담 직후 엑셀 파일에 서비스 내용을 기입 (원문 보관 목적)
2. **2차 기록**: 틈틈이 시간이 날 때 국가아동학대정보시스템에 동일 내용을 수동 입력

2차 기록 과정에서 발생하는 핵심 고통:
- 아동명 검색 → 사례 진입 → 서비스 제공 탭 → 신규등록 팝업 열기까지 **5~6회의 클릭**
- 팝업 내 드롭다운 선택, 텍스트 필드 복붙, 날짜 입력 등 **반복적 수동 작업**
- 한 건당 약 **5~10분**, 하루 수십 건 누적 시 **1~2시간 이상 소요**
- 야근의 직접적 원인이 되며, 본질적 업무(현장 출동, 심리치료 등)에 쓸 시간 잠식

### 1.3 비전
> "사회복지사들이 서류 작업이 아닌 **아동 보호**에 집중할 수 있는 세상을 만든다"

## 2. 린 캔버스 (Lean Canvas)

| 항목 | 내용 |
|---|---|
| **문제 (Problem)** | ① 엑셀 → 국가시스템 이중 입력에 하루 1~2시간 소모 ② 드롭다운·텍스트 필드 반복 작업으로 인한 피로·실수 ③ DB 입력 지연으로 인한 통계·보고 데이터 부정확 |
| **고객군 (Customer Segments)** | 전국 아동보호전문기관 소속 사회복지사 (약 75개 기관, 기관당 5~20명), 위탁 법인(굿네이버스, 초록우산 등) |
| **고유 가치 제안 (UVP)** | 엑셀 파일 하나로 국가시스템 DB 입력을 **1시간 → 5분**으로 줄여드립니다 |
| **솔루션 (Solution)** | 엑셀 파싱 + 브라우저 RPA 기반 로컬 데스크톱 앱 (데이터 비유출) |
| **채널 (Channels)** | 기관 내부 입소문, 전국 아보전 네트워크(협회/간담회), 위탁법인 본부 세일즈 |
| **수익원 (Revenue Streams)** | 기관별 월 구독료 (SaaS식) 또는 연간 라이선스 + 유지보수 계약 |
| **비용 구조 (Cost Structure)** | 개발 인건비, Supabase(인증/라이선스 관리), 고객 지원, 보안 인증 비용 |
| **핵심 지표 (Key Metrics)** | 기관 도입 수, 활성 사용자 수, 건당 입력 시간 단축률, 월 처리 건수 |
| **경쟁 우위 (Unfair Advantage)** | 현직 내부자로서의 도메인 이해 + 실제 사용자 피드백 루프 + 경쟁자 부재 시장 |

## 3. 타겟 사용자 정의

### 3.1 주요 페르소나

#### 페르소나 A: 김수진 사회복지사 (28세, 경력 3년)
- **소속**: 대전광역시아동보호전문기관
- **역할**: 아동학대 사례 상담 및 서비스 제공
- **일과**: 오전 현장 출동/상담 → 오후 기록 업무 → 퇴근 후 밀린 DB 입력
- **고통**: "상담 끝나고 PC 앞에 앉으면 또 시작이에요. 엑셀에 쓴 거 똑같이 시스템에 치기만 하면 되는데, 드롭다운 하나하나 누르고 복붙하고... 3명 거 하면 한 시간은 넘어요. 진짜 이거 자동으로 되면 소원이 없겠다..."
- **기술 수준**: 엑셀 중급, IT 도구 사용에 거부감 없음, 프로그래밍 경험 없음

#### 페르소나 B: 박영호 팀장 (42세, 경력 15년)
- **소속**: 전북 아동보호전문기관 사례관리팀장
- **역할**: 팀원 관리, 복지부 보고 데이터 취합
- **고통**: "팀원들이 시스템 입력을 밀려서 월말에 몰아서 하니까 통계가 지저분해요. 실적 보고할 때마다 수기로 맞춰야 하고..."
- **니즈**: 팀원들의 DB 입력 지연 해소, 실시간에 가까운 데이터 반영

#### 페르소나 C: 이정은 기관장 (50세)
- **역할**: 예산 집행 및 운영 총괄, 위탁 법인 보고
- **의사결정 기준**: 비용 대비 효과(ROI), 규정 준수(컴플라이언스), 직원 복지
- **우려**: "개인정보 유출 문제는 없나요? 보안 감사에 걸리면 안 되는데..."

### 3.2 사용 상황 (Context of Use)
| 상황 | 빈도 | 세부 |
|---|---|---|
| 상담 후 DB 일괄 입력 | 매일 오후 | 당일 진행한 3~10건의 서비스를 퇴근 전 몰아서 입력 |
| 월말 일괄 정리 | 월 1회 | 밀렸던 건들을 한꺼번에 소화 (20~50건) |
| 긴급 보고 대응 | 비정기 | 복지부/법인 요청 시 즉시 데이터 반영 필요 |

| P4 | 동명이인 아동 혼동 위험 | 🟡 Medium | 생년월일로 구분 (수동) |

| P6 | 단순 서류 업무에 시간을 빼앗겨 아동 상담에 집중 불가 | 서비스 질 하락, 번아웃 |

## 5. 서비스 핵심 기능

### 5.1 MVP (최소 기능 제품) — Phase 1

| 기능 | 설명 | 우선순위 |
|---|---|---|
| **엑셀 파일 로드** | 표준 템플릿에 맞춘 `.xlsx` 파일을 드래그 앤 드롭으로 불러오기 | P0 |
| **데이터 미리보기** | 불러온 엑셀 데이터를 테이블 형태로 보여주고, 입력 전 확인 | P0 |
| **브라우저 연결** | 이미 로그인된 크롬 브라우저(디버깅 모드)에 연결 | P0 |
| **단일 건 자동 입력** | 팝업창의 각 필드(드롭다운, 텍스트, 날짜)에 자동 기입 | P0 |
| **진행 상황 표시** | 현재 몇 건 중 몇 건 완료인지 프로그레스 바로 시각화 | P1 |
| **에러 로깅** | 입력 실패 건은 스킵하고 로그 파일로 기록 | P1 |

### 5.2 확장 기능 — Phase 2

| 기능 | 설명 | 우선순위 |
|---|---|---|
| **자동 네비게이션** | 아동명 검색 → 사례 진입 → 서비스 제공 탭 → 신규등록까지 자동 탐색 | P1 |
| **다건 연속 처리** | 엑셀의 여러 행을 순차적으로 반복 입력 (Loop) | P1 |
| **동명이인 처리** | 검색 결과가 2명 이상일 때 사용자에게 수동 선택 요청 후 대기 | P1 |
| **명시적 대기** | 시스템 로딩 지연 시 에러 없이 대기 (Explicit Wait) | P1 |

### 5.3 고도화 기능 — Phase 3

| 기능 | 설명 | 우선순위 |
|---|---|---|
| **로그인/라이선스 관리** | Supabase 인증 기반, 기관별 구독 관리 | P2 |
| **자동 업데이트** | 새 버전 출시 시 앱 내에서 원클릭 업데이트 | P2 |
| **입력 결과 리포트** | 성공/실패/스킵 건수를 PDF 또는 엑셀로 리포트 생성 | P2 |
| **다중 서비스 유형 지원** | 서비스 제공 외 다른 탭(상담 기록 등) 입력 자동화 확장 | P3 |

## 6. 입력 데이터 필드 매핑

국가아동학대정보시스템의 '서비스 제공 > 신규등록' 팝업에서 입력해야 하는 필드:

| 엑셀 컬럼 | 시스템 필드 | 입력 방식 | 비고 |
|---|---|---|---|
| 서비스 유형 | 제공 서비스 유형 | 드롭다운 선택 | 코드 매핑 필요 |
| 제공 장소 | 제공 장소 | 드롭다운 선택 | |
| 관련자 구분 | 관련자 (아동/대상자/가족구성원) | 드롭다운 또는 라디오 | |
| 서비스 시작일 | 서비스 제공 기간 (시작) | 날짜 입력 | 캘린더 팝업 가능 |
| 서비스 종료일 | 서비스 제공 기간 (종료) | 날짜 입력 | |
| 소요 시간 | 서비스 제공 시간 | 텍스트/숫자 | |
| 담당 상담원 | 상담원 | 드롭다운 또는 검색 | |
| 제공 내용 | 제공내용 | 텍스트 영역 (Textarea) | 가장 긴 필드 |

> ⚠️ **실제 필드명과 입력 방식은 시스템 팝업의 HTML 분석 후 확정 필요**

## 7. 유저 케이스 (Use Cases)

### UC-1: 일반 일괄 입력 (Happy Path)
```
사전 조건: 상담원이 엑셀 파일에 당일 서비스 5건을 기입 완료
1. 상담원이 Linker 프로그램을 실행한다
2. 국가시스템이 이미 로그인된 크롬 브라우저를 확인한다
3. 엑셀 파일을 Linker 화면에 드래그 앤 드롭한다
4. Linker가 데이터를 파싱하여 미리보기 테이블을 보여준다
5. 상담원이 내용을 확인하고 [자동 입력 시작] 버튼을 클릭한다
6. Linker가 아동명을 검색 → 사례 진입 → 서비스 제공 → 신규등록 순으로 탐색한다
7. 팝업이 뜨면 엑셀 데이터를 각 필드에 자동 기입한다
8. [저장] 클릭 후 다음 아동으로 이동한다
9. 5건 완료 후, 결과 요약(성공 5건, 실패 0건)을 보여준다
사후 조건: 5건의 DB가 국가시스템에 정상 등록됨
```

### UC-2: 동명이인 발생 시
```
사전 조건: 엑셀에 '김민준' 아동의 서비스 기록 존재
1~6. UC-1과 동일
7. 아동명 검색 결과가 2명 이상 나옴
8. Linker가 자동 입력을 일시 정지하고 경고 팝업을 띄움
     "[⚠️ 동명이인 2명 발견] 직접 올바른 아동을 선택해주세요"
9. 상담원이 직접 올바른 아동을 클릭함
10. Linker가 이어서 나머지 과정을 자동 수행함
사후 조건: 올바른 아동의 사례에 DB가 등록됨
```

### UC-3: 네트워크/시스템 오류 발생 시
```
사전 조건: 5건 중 3건째 입력 중 국가시스템 서버 타임아웃 발생
1. Linker가 타임아웃을 감지함
2. 해당 건을 '실패'로 로그에 기록함
3. 사용자에게 "3번째 건(김OO) 입력 실패 — 시스템 응답 없음" 알림
4. 자동으로 4번째 건으로 넘어가 입력 재개함
5. 최종 결과: 성공 4건, 실패 1건 (실패 건 목록 제공)
사후 조건: 실패 건은 별도 재시도 가능
```

## 8. 유저 저니맵 (User Journey Map)

### Phase 1: 인지 (Awareness)
```
상담원 김수진: "오늘도 DB 입력하다가 야근이네... 이걸 자동으로 해주는 건 없나?"
              ↓
        동료로부터 Linker 소개받음 (입소문)
              ↓
        "우리 기관 윤수 씨가 만든 거래. 한번 써볼래?"
```

### Phase 2: 도입 (Onboarding)
```
        Linker 설치 파일(.exe) 전달받음
              ↓
        실행 → 간단한 사용법 안내 화면 (3단계)
              ↓
        ① 크롬 브라우저를 디버깅 모드로 실행하세요
        ② 국가시스템에 로그인해 두세요
        ③ 엑셀 파일을 여기에 놓으세요
              ↓
        "이게 끝이야? 진짜 간단하다!"
```

### Phase 3: 첫 사용 (First Use)
```
        엑셀 파일 드래그 앤 드롭
              ↓
        미리보기에서 데이터 확인 → "맞네, 이거 오늘 상담한 거"
              ↓
        [시작] 클릭 → 브라우저가 자동으로 움직이기 시작
              ↓
        😲 "와... 진짜 알아서 치네?! 드롭다운도 알아서 누르고?!"
              ↓
        5건 완료 알림: "전부 성공입니다! (소요시간: 2분 30초)"
              ↓
        😭 "1시간 걸리던 게 2분이요?! 이거 진짜 혁명이다..."
```

### Phase 4: 일상 사용 (Habitual Use)
```
        매일 오후 4시, 상담 기록 정리 후 Linker 실행
              ↓
        "이제 이거 없으면 일 못 해요"
              ↓
        동료에게 추천 → 파급 효과
              ↓
        기관장도 인지 → "이거 우리 기관 전체가 써야 하는 거 아니야?"
```

### Phase 5: 확산 (Expansion)
```
        전국 아동보호전문기관 네트워크를 통한 입소문
              ↓
        타 기관에서 도입 문의
              ↓
        유료 라이선스(월 구독 or 연간 계약) 전환
              ↓
        🎯 GTM 성공
```

## 9. 기술 아키텍처

### 9.1 시스템 구성도

```
┌───────────────────────────────────────────────────────────┐
│                    사용자 PC (로컬)                         │
│                                                           │
│  ┌──────────────┐     ┌──────────────┐                    │
│  │  엑셀 파일    │────▶│   Linker     │                    │
│  │  (.xlsx)      │     │  (Python     │                    │
│  └──────────────┘     │   Desktop    │                    │
│                       │   App)       │                    │
│                       │              │──── Selenium ────▶ │
│                       │  - Pandas    │  or PyAutoGUI      │
│                       │  - GUI       │                    │
│                       │    (PyQt/    │     ┌────────────┐ │
│                       │     CTk)     │     │ Chrome     │ │
│                       └──────┬───────┘     │ (디버깅    │ │
│                              │             │  모드)     │ │
│                              │             │            │ │
│                              │             │  국가아동   │ │
│                              │             │  학대정보   │ │
│                              │             │  시스템     │ │
│                              │             └────────────┘ │
│                              │                            │
└──────────────────────────────┼────────────────────────────┘
                               │ (인증/라이선스 확인만)
                               ▼
                    ┌──────────────────┐
                    │    Supabase      │
                    │  (클라우드)       │
                    │  - 사용자 인증    │
                    │  - 라이선스 관리  │
                    │  - 사용 통계     │
                    └──────────────────┘
```

### 9.2 핵심 설계 원칙

1. **데이터 비유출 (Zero Data Leakage)**: 아동 개인정보는 절대 외부 서버로 전송하지 않음. 모든 처리는 사용자 PC 내에서만 수행
2. **기존 로그인 활용**: 보안 프로그램(키보드 보안 등)을 우회하기 위해 이미 로그인된 브라우저에 연결하는 방식 채택
3. **안전 우선 (Safety First)**: 동명이인, 로딩 지연, 네트워크 오류 시 자동 중단 후 사용자 확인 요청
4. **최소 침습**: 국가시스템의 API를 호출하는 것이 아니라 UI를 사람처럼 조작하므로, 시스템에 부하를 주지 않음

## 12. 개발 로드맵 (8주)

| 주차 | 마일스톤 | 핵심 산출물 |
|---|---|---|

## 13. 수익 모델

### Option A: 기관별 월 구독 (SaaS식)
- **Basic**: 월 5만 원 (사용자 3명까지, 월 100건 입력)
- **Standard**: 월 10만 원 (사용자 10명, 무제한 입력)
- **Enterprise**: 월 20만 원 (무제한 사용자, 우선 지원, 맞춤 설정)

### Option B: 연간 라이선스 + 유지보수
- 연간 100만 원 (기관 전체) + 유지보수 월 5만 원
- 정부·공공기관은 '연간 계약'에 익숙하므로 예산 편성에 유리

### Option C: 위탁법인 본부 일괄 계약
- 굿네이버스 본부와 계약 → 산하 전체 기관에 배포
- 기관당 단가 인하 but 볼륨 확보

> 💡 **추천**: 복무 기간 중 무료 배포로 팬덤 확보 → 소집해제 후 Option B or C로 유료 전환
