# 🚀 Linker(링커) — GTM 전략 v2.0 (Chrome 확장 프로그램 포지셔닝)

> **"설치 한 번, 영원히 편한 DB 입력" — 브라우저 확장 프로그램 기반 업무 자동화 서비스**

---

## 1. 포지셔닝 전환: v1 → v2

| 항목 | v1 (Python RPA) | v2 (Chrome 확장 프로그램) |
|---|---|---|
| **설치 방법** | Python 설치 + 터미널 명령어 | 확장 프로그램 클릭 한 번 |
| **실행 방법** | 디버깅 모드 Chrome 별도 실행 | 평소 쓰던 Chrome/Edge 그대로 |
| **업데이트** | 수동 파일 교체 | 자동 업데이트 |
| **IT 지식 필요** | 높음 | 없음 |
| **배포 방식** | 파일 전달 | Chrome 웹 스토어 / Edge 스토어 링크 하나 |
| **세일즈 장벽** | "설치가 너무 복잡해요" | "링크 클릭하면 설치돼요" |

> 💡 **핵심 변화**: 기술 제품에서 **생활 도구**로. 상담원이 카카오톡 설치하듯 Linker를 설치한다.

## 2. GTM 핵심 전략: "Inside-Out Expansion" (유지)

```
[Phase 0] 대전 기관 내부 → 무료 설치 테스트 (확장 프로그램 링크 공유)
     ↓
[Phase 1] 대전 기관 전체 도입 → 성공 사례(Case Study) 확보
     ↓
[Phase 2] 같은 위탁법인 산하 타 지역 기관 → 법인 본부 일괄 배포
     ↓
[Phase 3] 전국 아동보호전문기관 → 스토어 등록 + 유료 전환
     ↓
[Phase 4] 타 복지 영역(요양원, 장애인복지관 등) → 수평 확장
```

**v2에서 달라지는 점**: Phase 2부터 법인 본부 IT팀이 **"링크 하나"로 산하 전 기관에 일괄 배포** 가능. 확산 속도가 v1 대비 10배 빠름.

## 3. 시장 규모 추정 (TAM / SAM / SOM)

### TAM (전체 시장)
- 정부 시스템에 DB를 입력하는 전국 사회복지시설 **약 45,000개소**
- 아동보호, 노인요양, 장애인복지, 정신건강 등 복지 전 영역

### SAM (접근 가능 시장)
- 전국 아동보호전문기관: **약 75개 기관**, 상담원 약 750명
- 기관당 연간 라이선스 60만 원 기준: **SAM ≈ 4,500만 원/년**

### SOM (초기 확보 가능 시장)
- 복무 기간 중 직접 접촉 가능한 기관: 대전 + 같은 위탁법인 산하 **5~10개 기관**
- 소집해제 후 1년 내 도입 목표: **20개 기관**
- **SOM ≈ 1,200만 원/년** (기관당 60만 원 × 20개)

> ⚠️ 아동보호전문기관만으로는 시장 규모가 크지 않음. 그러나 **검증된 성공 사례**를 기반으로 타 복지 도메인(요양원 2만 개+)으로 확장하면 **수십억 규모** 시장으로 점프 가능

## 4. 경쟁 환경 분석

### 4.1 직접 경쟁자

| 분류 | 현황 |
|---|---|
| 국가시스템 자체 개선 | 사회보장정보원이 직접 UI/UX를 개선할 가능성. 하지만 대규모 정부 시스템의 개선 속도는 극히 느림 (수년 단위) |
| 범용 RPA 도구 (UiPath 등) | 라이선스 비용이 비영리 기관 예산 대비 과도 (연 수천만 원), 비전문가가 쓰기 어려움 |
| 범용 브라우저 확장 (iMacros 등) | 존재하나 도메인 특화 설정이 필요해 비전문가 사용 불가 |

### 4.2 경쟁 우위 (MOAT)

| 우위 요인 | 설명 |
|---|---|
| **도메인 전문성** | 현직 내부자로서 시스템과 워크플로우를 체감 수준으로 이해 |
| **설치 장벽 제로** | 확장 프로그램 링크 하나로 설치 완료. 범용 RPA 대비 압도적 접근성 |
| **가격 경쟁력** | 범용 RPA 대비 1/10 이하 비용으로 도메인 특화 솔루션 제공 |
| **전환 비용** | 한번 도입되면 워크플로우에 깊이 통합되어 이탈하기 어려움 |
| **네트워크 효과** | 한 기관이 도입하면 같은 법인 산하 기관들에 자연스럽게 전파 |
| **경쟁자 부재** | 이 니치(Niche) 시장에 특화된 솔루션이 아직 존재하지 않음 |

## 5. 세일즈 전략

### 5.1 복무 기간 (남은 기간) — "무료 전사(Evangelist) 전략"

| 단계 | 액션 | 기대 효과 |
|---|---|---|
| **내부 증명** | 상담원 1~3명에게 확장 프로그램 링크 공유 → 무료 설치 | "이거 없으면 일 못 해" 반응 확보 |
| **기관장 보고** | 입력 시간 단축 데이터를 수치화하여 기관장에게 보고 | 내부 공식 승인 획득 |
| **성과 문서화** | Before/After 영상, 사용 후기, 시간 절감 데이터 수집 | 케이스 스터디 자산 확보 |
| **네트워크 심기** | 전국 아보전 간담회/연수에서 QR코드로 설치 시연 | 타 기관 관심 유발 |

### 5.2 소집해제 후 — 세일즈 타겟별 전략

#### 🎯 타겟 1: 개별 기관 상담원 (Bottom-Up)
- **메시지**: "엑셀 드래그 한 번으로 DB 입력 끝. 야근 없어집니다."
- **채널**: 사복넷, 복지로 등 사회복지사 커뮤니티, 카카오톡 단체방
- **전환 방식**: 무료 체험 → 기관 라이선스 전환 요청 유도

#### 🎯 타겟 2: 기관장 (ROI 중심)
- **메시지**: "상담원 1인당 월 X시간 절감 = 연 Y만 원 인건비 절감 효과"
- **채널**: 직접 방문 + 데이터 기반 1페이지 제안서
- **전환 방식**: 기관 라이선스 (연 60만 원) 계약

#### 🎯 타겟 3: 위탁법인 본부 IT팀 (Scale 중심)
- **메시지**: "산하 N개 기관에 링크 하나로 일괄 배포. 관리 비용 제로."
- **채널**: 법인 IT담당자 또는 사업부장 미팅
- **전환 방식**: 법인 패키지 계약 (기관당 연 40만 원 × N개)
- **핵심 포인트**: 확장 프로그램은 **IT팀이 별도 서버 관리 불필요** → 도입 장벽 최소화

#### 🎯 타겟 4: 보건복지부 / 사회보장정보원 (정책 중심)
- **메시지**: "현장 실무자의 시스템 사용 만족도 제고 + 데이터 입력 정확도 향상"
- **채널**: 정책 제안서, 사회복지 디지털 혁신 공모 사업
- **전환 방식**: 공식 협력 또는 정부 예산 지원

## 6. 계약 및 배포 방식 (확장 프로그램 특화)

### 6.1 배포 시나리오

```
[개인 상담원]
  → Chrome 웹 스토어 / Edge 스토어에서 직접 설치
  → 무료 체험 후 기관 라이선스 구매 요청

[기관 단위 도입]
  → 기관장이 라이선스 구매
  → Linker가 기관 도메인 이메일(@기관명.or.kr) 기준으로 인증
  → 상담원들은 링크 하나로 설치 후 기관 계정으로 로그인

[법인 본부 일괄 도입]
  → 법인 IT팀이 패키지 계약
  → 법인 본부가 산하 기관 전체에 설치 링크 + 가이드 공유
  → 개별 기관 별도 계약 불필요
```

### 6.2 계약 형태

| 계약 주체 | 계약 형태 | 특이사항 |
|---|---|---|
| 개별 상담원 | 개인 구독 (카드 결제) | 기관 예산 불필요, 즉시 전환 가능 |
| 기관 | 기관 라이선스 (세금계산서) | 전산운영비 또는 수용비 항목으로 집행 |
| 위탁법인 | 법인 패키지 (연간 계약서) | 볼륨 디스카운트, 전담 지원 포함 |

> 💡 **핵심**: 확장 프로그램은 서버 설치가 없으므로 **기관 IT팀의 보안 검토 부담이 낮음**. 기관 PC에 별도 프로그램을 설치하는 것이 아니라 브라우저 확장이기 때문에 보안 정책 통과가 상대적으로 쉬움.

## 8. 가격 전략 (Pricing)

### 8.1 비영리 기관의 지불 현실

> 💡 **핵심 인사이트**: 비영리 기관은 "비싸서 못 사는 것"이 아니라 "예산 항목에 없어서 못 사는 것"인 경우가 많다.

| 고려 사항 | 내용 |
|---|---|
| 연간 예산 편성 주기 | 보통 전년도 9~11월에 다음 해 예산 확정. 중간 도입 시 '전산운영비' 또는 '수용비' 항목에서 집행 가능한지 확인 필요 |
| 의사결정 속도 | 기관장 전결(소액) vs 법인 본부 승인(고액). 월 5만 원 수준이면 기관장 전결 가능성 높음 |
| 지불 방식 선호 | 월 구독보다 **연간 일시불** 선호 (정부 예산 집행 특성상) |

### 8.2 추천 가격안

| 플랜 | 가격 | 포함 내용 | 타겟 |
|---|---|---|---|
| **무료 체험** | 0원 / 1개월 | 전 기능, 월 30건 제한 | 검증 및 온보딩 |
| **기관 라이선스** | 연 60만 원 (월 5만 원) | 무제한 사용자·건수, 이메일 지원 | 소규모 기관 |
| **법인 패키지** | 기관당 연 40만 원 (5개 기관 이상) | 볼륨 디스카운트 + 전담 지원 | 위탁법인 본부 |

## 9. 핵심 인사이트 & 제언

### 💡 인사이트 1: "설치가 세일즈다"
확장 프로그램의 가장 강력한 무기는 **설치 장벽이 없다**는 것입니다. 상담원이 링크를 받아 클릭하면 30초 안에 설치 완료. 이 경험 자체가 "이 서비스는 어렵지 않다"는 신뢰를 만듭니다. 세일즈 미팅에서 노트북 열고 즉석 시연 → 즉석 설치까지 가능.

### 💡 인사이트 2: "IT팀의 언어로 말하라"
법인 본부 IT팀을 설득할 때 핵심 메시지:
- **"서버 설치 없음"** — 기관 인프라에 아무것도 추가하지 않음
- **"브라우저 권한만 사용"** — 기관 PC 보안 정책에 영향 없음
- **"중앙 관리 가능"** — 법인 본부에서 산하 기관 라이선스 일괄 관리

### 💡 인사이트 3: "기관장의 언어로 말하라"
- **"DB 입력 적시성 향상으로 복지부 실적 보고 정확도 제고"**
- **"상담원 1인당 사례 관리 가능 건수 확대"**
- **"직원 이직률 감소를 통한 조직 안정화"**

### 💡 인사이트 4: "정부 혁신 사업 공모에 도전하라"
보건복지부, 한국사회보장정보원, 지자체의 **'사회복지 현장 디지털 혁신'** 공모 사업에 Linker를 제안:
- 개발비를 정부 예산으로 지원받을 수 있음
- 공식적인 도입 채널이 열림
- 사회보장정보원과의 공식 협력 관계 구축 가능

### 💡 인사이트 5: "Chrome 웹 스토어 등록이 신뢰의 증거"
스토어에 등록된 확장 프로그램은 Google/Microsoft의 검토를 통과했다는 의미. 기관장이나 IT팀에게 **"구글이 검증한 프로그램"**이라는 신뢰 신호로 작용. 개인이 만든 파일을 PC에 설치하는 것보다 심리적 장벽이 훨씬 낮음.

## 10. 엑셀 포맷 문제 해결 전략

> 상담원마다 다른 엑셀 양식을 쓰는 문제 → **컬럼 매핑 UI**로 해결

```
[v1 PoC]  우리 포맷 엑셀만 인식 (현재)
     ↓
[v1 정식] 컬럼 매핑 UI: 상담원이 "이 열이 제목이에요" 한 번 지정 → 저장
     ↓
[v2]      AI 자동 매핑: 컬럼명 보고 자동 추론 → 상담원 확인만
```

상담원이 기존에 쓰던 엑셀을 그대로 올리고, 처음 한 번만 매핑하면 이후엔 자동 인식. **포맷 변경 요구 없음.**

## 11. 시스템 및 운영 아키텍처 (System & Operations Architecture)

### 11.1 사용자 인증 및 계정 체계

| 구분 | **Free (체험판)** | **Pro (기관 인증 회원)** |
|---|---|---|
| **타겟 유저** | 신규 사용자, 도입 검토 중인 상담원 | 정식 도입 기관 소속 상담원 |
| **인증 방식** | Google 소셜 로그인 (개인 계정) | **기관 도메인 이메일 인증** (@gnk.or.kr 등) |
| **데이터 저장** | 로컬 브라우저 저장소 (IndexedDB) | 클라우드 동기화 (설정값, 매핑 규칙) |
| **사용량 제한** | **월 30건 제한** | **무제한** |
| **기능 차이** | 기본 기능만 제공 | 팀 공유(매핑 규칙 공유), 우선 기술지원, 관리자 대시보드 |

### 11.2 기관 도메인 인증 프로세스 자동화

> **"관리자 개입 없는 자동 소속 처리"**

1. **도메인 화이트리스트 구축**: 주요 위탁법인 및 아동보호전문기관의 도메인(@sc.or.kr, @gnk.or.kr, @holt.or.kr 등)을 DB화.
2. **인증 흐름**:
   - `사용자`: 회원가입 시 업무용 이메일 입력
   - `시스템`: 인증 코드 발송 (SMTP)
   - `사용자`: 코드 검증 완료
   - `시스템`: 이메일 도메인을 확인하여 **즉시 해당 기관(Tenant) 소속으로 자동 분류**
3. **라이선스 자동 적용**: 해당 기관이 이미 유효한 라이선스를 보유 중이라면, 별도 결제 없이 즉시 Pro 권한 부여.

### 11.3 데이터 분리 및 보안 정책 (Privacy by Design)

> **핵심 원칙: "Linker는 파이프라인일 뿐, 민감 데이터를 저장하지 않는다"**

가장 큰 보안 리스크인 '아동 학대 관련 민감 정보' 유출 가능성을 원천 차단합니다.

- **Zero-Knowledge Architecture**:
  - 엑셀 파일 내용은 **사용자의 브라우저 메모리(Client-Side)** 안에서만 파싱되고 처리됩니다.
  - 파싱된 민감 데이터(이름, 생년월일, 학대 내용 등)는 **절대로 Linker 서버로 전송되거나 저장되지 않습니다.**
- **서버 저장 데이터 최소화**:
  - 사용자 계정 정보(이메일, 이름)
  - 라이선스 및 결제 정보
  - 사용량 메타데이터(처리 건수, 성공/실패 여부, 타임스탬프)
  - **(중요)** 사용자 정의 엑셀 매핑 규칙 (컬럼명 설정값만 저장, 실제 데이터 내용 아님)

### 11.4 결제 및 정산 정책

- **개인 카드 결제 (B2C)**
  - PG사(Toss Payments 등) 연동을 통한 월 정기 구독.
  - 개인 카드로 결제하고 소액 지출 증빙용 영수증 자동 발급.
- **기관 세금계산서 발행 (B2B)**
  - 서비스 내 '견적서 요청' 버튼 제공 → PDF 견적서 자동 발송.
  - 무통장 입금 확인 후 라이선스 키 발급 또는 기관 도메인 전체 활성화.
  - **선청구(Pre-bill)** 방식: 예산 집행 확정을 위해 세금계산서를 먼저 발행하고, 입금 기한을 주는 방식 지원 (공공기관 특성 반영).
