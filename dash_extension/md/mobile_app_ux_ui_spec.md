# Dash Mobile App: UX/UI 설계 사양서 (v1.0)

Dash 모바일 앱은 상담원의 '현장 업무'를 책임지는 출발점입니다. 상담 직후 가장 생생한 기억을 단 1분 만에 데이터화하는 것에 집중합니다.

---

## 1. 디자인 원칙 (Design Principles)
- **One-Handed Operation**: 현장에서 스마트폰을 한 손으로 쥐고도 주요 DB 항목을 탭(Tap)할 수 있는 하단 중심 레이아웃.
- **High-Density Chips**: 드롭다운 대신 모든 옵션을 화면에 노출하여 클릭 횟수를 최소화.
- **Trust & Security**: E2EE 암호화 상태를 시각적으로 보여주어 유저에게 심리적 안전감을 제공.

---

## 2. 주요 화면 구성 (Screen Flow)

### 2.1. 홈 화면 (Case List)
- **기능**: 오늘 입력해야 할 대상자 리스트 노출.
- **UI 요소**: 
  - 검색 바 (아동 이름 검색)
  - 미완료/진행 중/완료 탭 구분
  - 각 카드별 '마지막 상담일' 정보 표시

### 2.2. 현장 기록 화면 (Field Recording) - **핵심**
- **대상자 정보 (Header)**: 아동명, 상담 차수, 현재 시간.
- **DB 필드 셋팅 그리드 (The Chips)**:
  - **제공장소**: [가정] [사회복지시설] [학교] [병원] (2x2 그리드)
  - **제공방법**: [방문] [내방] [전화] (3x1 그리드)
  - **이동시간**: [10분] [20분] [30분] [60분] (4x1 스크롤)
- **상담 내용 입력 (Notes)**: 
  - 큰 텍스트 영역
  - (미래 확장) 음성 녹음 아이콘
- **Sync & Share 바 (Bottom Action)**: 
  - [저장 및 동기화] 대형 버튼 (FAB 또는 하단 고정 바)

---

## 3. 정보 계층 구조 (Information Hierarchy)

1. **[Level 1] 누구인가?**: 아동명 (Child A)
2. **[Level 2] 어디서, 어떻게 했는가?**: DB 필드 (Location, Method, Time) - 클릭만으로 완료
3. **[Level 3] 무엇을 했는가?**: 상담 내용 (Notes) - 직접 타이핑 또는 음성
4. **[Level 4] 어떻게 보낼 것인가?**: 공유 링크 생성 및 메신저 전송

---

## 4. 핵심 UX 인터랙션 (Key Interactions)

### 4.1. 스마트 칩(Smart Chips) 멀티 선택
- 드롭다운 메뉴를 여는 행위 없이, 화면에 펼쳐진 버튼을 누르는 즉시 선택 완료.
- 선택된 항목은 Dash의 브랜드 컬러(Solid Blue)로 강조되어 시각적 확인이 즉각적임.

### 4.2. E2EE 락킹 애니메이션 (Locking Animation)
- [저장] 버튼 클릭 시, 텍스트가 암호화된 기호로 바뀌어 상자 안으로 들어가는 듯한 애니메이션 제공.
- "데이터가 안전하게 자물쇠로 잠겼습니다"라는 메시지를 통해 보안에 대한 신뢰 형성.
