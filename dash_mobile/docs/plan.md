# Dash(대시) — 서비스 기획서 (PRD)

> **아동보호전문기관 사회복지사를 위한 국가아동학대정보시스템 DB 자동 입력 솔루션**
> 최종 업데이트: 2026-05-30 (v3.5.x 현행화)

---

## 1. 서비스 개요

### 1.1 한 줄 정의
> 현장에서 모바일 앱으로 작성한 상담 기록을 사무실 PC의 Chrome 확장 프로그램으로 **1초 만에 국가아동학대정보시스템에 자동 주입**하는 통합 워크플로우 플랫폼

### 1.2 제품 정보
| 항목 | 내용 |
|---|---|
| **제품명** | Dash · 초간단 아동학대정보시스템 DB 입력기 |
| **형태** | ① Flutter 모바일 앱 (iOS/Android) + ② Chrome/Edge 확장 프로그램 (Manifest V3) + ③ 클라우드 중계 서버 |
| **확장 버전** | 3.5.x (Chrome Web Store 배포 중) |
| **서버** | https://dash.qpon (Express.js + MySQL, Railway 호스팅) |
| **배포** | Chrome Web Store (확장 프로그램), App Store/Play Store (모바일, 준비 중) |
| **개인정보처리방침** | https://roddbstn.github.io/Dash-/ |

### 1.3 배경 및 문제 인식
전국 아동보호전문기관 사회복지사들은 아동학대 상담 후 DB 기록 과정에서 다음 페인 포인트를 겪는다:

1. **데이터 파편화**: 현장 메모(클로바노트) → 이동 중 정리(메모장) → 상급자 컨펌(메신저) → DB 입력(국가시스템)까지 동일 내용을 4~5번 반복
2. **시스템 불안정 공포**: 느리고 튕기는 국가시스템에서 작업하다 데이터 유실
3. **반복 드롭다운 작업**: 매번 수동으로 드롭다운을 하나씩 선택하는 신체적 피로
4. **다중 사례 처리**: 여러 아동 창을 동시에 열어놓고 각각 따로 입력하는 번거로움

**핵심 고통**: 한 건당 약 5~10분, 하루 수십 건 누적 시 1~2시간 이상 야근의 직접적 원인

### 1.4 비전
> "사회복지사들이 서류 작업이 아닌 **아동 보호**에 집중할 수 있는 세상을 만든다"

## 2. 워크플로우 변화상 (AS-IS vs TO-BE)

| 단계 | AS-IS (파편화된 프로세스) | TO-BE (Dash 통합 파이프라인) |
|---|---|---|
| **1. 현장 기록** | 클로바노트 녹음 또는 수기 메모 | **Dash 모바일 앱**에서 칩 선택 + 메모 작성 |
| **2. 데이터 정제** | 이동 중 메모장에 직접 요약 재정리 | *(Phase 2)* AI 자동 정제 |
| **3. 협업/컨펌** | 메신저로 정제본 전송 → 개별 피드백 | Dash **공유 링크** 원클릭 전송 → Reviewer 웹에서 열람 |
| **4. DB 기입** | 최종본 복사 → 느린 시스템에 직접 타이핑 | PC 복귀 후 **Dash Extension**에서 리스트 선택 → '주입' 버튼 |
| **사용 도구 수** | 4~5개 | **단 1개 (Dash)** |

### 6.3 사용 권한 (최소 권한 원칙)

| 권한 | 용도 |
|---|---|
| `storage` | PIN vault, 캐시 토큰, UI 설정을 로컬에 저장 |
| `tabs` | 열려 있는 시스템 탭(창)들의 ID/URL 조회 |
| `sidePanel` | 사이드 패널 UI 제공 |
| `identity` | Google OAuth (chrome.identity.launchWebAuthFlow) |
| `content_scripts` | ncads.go.kr, localhost에서만 DOM 제어 스크립트 자동 주입 |

### 6.4 핵심 설계 원칙

1. **E2EE (End-to-End Encryption)**: 데이터는 모바일 기기에서 AES-256으로 암호화 후 서버로 전송. 서버는 암호화된 blob만 보관하며 원문 접근 불가
2. **PIN Vault**: 암호화 키를 PIN으로 잠근 vault에 보관. 모바일과 Extension 모두 동일한 PIN 사용
3. **완전 독립 작업**: 시스템 로그인 전에 Dash 앱에서 모든 DB 항목을 사전 완성. 시스템 튕김으로 인한 데이터 유실 위험 0%
4. **최소 시스템 접촉**: 국가시스템 팝업 창에서 주입 완료까지 최대 1초 (Magazine Injection)

### UC-1: 현장 기록 → 사무실 주입 (Happy Path — 엔드투엔드)

```
사전 조건: 상담원이 외부 현장에서 상담 완료

[현장 — 모바일 앱]
1. Dash 앱 실행 → 해당 사례 선택
2. 제공장소/방법/서비스유형/날짜를 칩 버튼으로 3초 내 선택
3. 서비스 내용 및 상담원 소견 메모 작성
4. [저장 및 동기화] 버튼 → 기기 내 AES-256 암호화 후 서버 전송
5. (선택) [공유 링크 생성] → 카카오톡/메신저로 상급자에게 전송

[사무실 — PC 확장 프로그램]
6. Chrome에서 국가아동학대정보시스템 접속 → 해당 아동 팝업창 열기
7. Dash Extension 사이드 패널 열기
8. "나의 DB" 탭에서 해당 레코드 확인 ("Secured via E2EE" 표시)
9. [주입] 버튼 클릭 → 1초 내 모든 필드 자동 기입
10. 대상자/서비스 제공자 수동 선택 후 시스템 [저장] 클릭

사후 조건: DB 기입 완료. 레코드는 "이전 기록" 탭으로 이동
```

### UC-2: 공유 링크 리뷰 워크플로우

```
사전 조건: 상담원이 UC-1의 5단계에서 공유 링크를 생성하여 대리님에게 전송

[대리님 — 리뷰어 웹]
1. 메신저 공유 링크 클릭 → Reviewer 웹 열림
2. DB 내용(서비스 내용, 소견, 메타정보) 확인
3. 수정이 필요한 경우: [수정하기] 클릭 → 이름 입력 인증
4. 서비스 내용 및 소견 텍스트 편집 후 [저장]
5. (향후 Phase 2) 확장 프로그램에 실시간 반영

[간사 — 모바일 앱]
6. 공유 DB 수신 알림 → 앱에서 수정 내용 확인 후 최종 저장
```

### UC-3: 신규 사용자 온보딩

```
1. Chrome Web Store에서 'Dash' 검색 → [Chrome에 추가]
2. 확장 프로그램 아이콘 클릭 → 사이드 패널 열림
3. [Google 계정으로 로그인] 클릭 → Google OAuth 인증
4. 모바일 앱에서 먼저 가입한 계정이어야 로그인 가능
   (미가입 시: "DASH 모바일 앱에서 가입 후 이용해주세요" 안내)
5. PIN 4자리 입력 → 메인 화면 진입
```
