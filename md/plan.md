# 📋 Dash(대시) — 서비스 기획서 (PRD)

> **아동보호전문기관 사회복지사를 위한 국가아동학대정보시스템 DB 자동 입력 솔루션**
> 최종 업데이트: 2026-02-27

---

## 1. 서비스 개요

### 1.1 한 줄 정의
> 엑셀 파일의 상담 기록을 브라우저 사이드 패널에 올리면, 열려 있는 여러 시스템 창에 **클릭 한 번으로 동시 자동 입력**해주는 **Chrome/Edge 브라우저 확장 프로그램**

### 1.2 제품 정보
| 항목 | 내용 |
|---|---|
| **제품명** | Dash · 초간단 아동학대정보시스템 DB 입력기 |
| **브랜드 로고** | dA (db Automate) |
| **형태** | Chrome/Edge 브라우저 확장 프로그램 (Manifest V3) |
| **배포** | Chrome Web Store (심사 제출 완료, 2026.02.27 기준) |
| **버전** | 2.0 |
| **구동 방식** | 완전 로컬 클라이언트 사이드 (서버 없음, 데이터 외부 전송 없음) |
| **개인정보처리방침** | https://roddbstn.github.io/Dash-/ |

### 1.3 배경 및 문제 인식
전국 아동보호전문기관의 사회복지사(상담원)들은 아동학대 사례에 대한 상담·서비스를 수행한 후, 시스템에 기록하는 과정에서 다음과 같은 **실무적 페인 포인트**를 겪고 있다:

1. **메모 업무의 파편화**: 외근 시 동행자가 **클로바 노트** 등에 적어준 메모를 보고 다시 시스템에 옮겨 적는 비효율 (단순 전사가 아닌 '판단'과 '기입'이 섞인 노동)
2. **고정된 서비스의 반복 입력**: 초기면담 시 제공하는 의무 교육(아동권리교육, 학대예방교육 등)과 같이 **항상 고정된 내용**을 매번 수동으로 드롭다운을 하나씩 선택하며 입력해야 하는 번거로움
3. **시스템의 복잡한 UI**: 아동명 검색 → 사례 진입 → 서비스 제공 탭 → 신규등록 팝업 열기까지 거쳐야 하는 **다단계 클릭**
4. **다중 사례 처리**: 여러 아동의 팝업창을 동시에 열어놓고 **각각 따로 입력**해야 하는 번거로움

핵심 고통:
- 팝업 내 드롭다운 선택, 텍스트 필드 복붙, 날짜 입력 등 **단순 반복적 수동 작업**
- 한 건당 약 **5~10분**, 하루 수십 건 누적 시 **1~2시간 이상 소요**
- 야근의 직접적 원인이 되며, 본질적 업무(아동 상담, 치료)에 쓸 시간 잠식

### 1.4 비전
> "사회복지사들이 서류 작업이 아닌 **아동 보호**에 집중할 수 있는 세상을 만든다"

## 2. 린 캔버스 (Lean Canvas)

| 항목 | 내용 |
|---|---|
| **문제 (Problem)** | ① 엑셀 → 국가시스템 이중 입력에 하루 1~2시간 소모 ② 드롭다운·텍스트 필드 반복 작업으로 인한 피로·실수 ③ 여러 아동 창을 동시에 처리해야 할 때 혼동 발생 |
| **고객군 (Customer Segments)** | 전국 아동보호전문기관 소속 사회복지사 (약 75개 기관, 기관당 5~20명), 위탁 법인(굿네이버스, 초록우산 등) |
| **고유 가치 제안 (UVP)** | 엑셀 파일 하나로 여러 시스템 창에 **동시에** DB 입력, **1시간 → 5분** |
| **솔루션 (Solution)** | Chrome/Edge 사이드 패널 기반 확장 프로그램 (DOM 직접 제어, 완전 로컬 구동) |
| **채널 (Channels)** | Chrome Web Store 무료 배포, 기관 내부 입소문, 전국 아보전 네트워크(협회/간담회) |
| **수익원 (Revenue Streams)** | Phase 1: 무료 배포 (팬덤 확보) → Phase 2: 기관별 프리미엄 구독 또는 위탁법인 B2B |
| **비용 구조 (Cost Structure)** | 개발 인건비 (1인 개발), Chrome Web Store 등록비 ($5 일회성) |
| **핵심 지표 (Key Metrics)** | 설치 수, 활성 사용자 수, 건당 입력 시간 단축률, 월 처리 건수, 스토어 평점 |
| **경쟁 우위 (Unfair Advantage)** | 현직 내부자로서의 도메인 이해 + 실제 사용자 피드백 루프 + 경쟁자 부재 시장 |

## 3. 타겟 사용자 정의

### 3.1 주요 페르소나

#### 페르소나 A: 김수진 사회복지사 (28세, 경력 3년)
- **소속**: 대전광역시아동보호전문기관
- **역할**: 아동학대 사례 상담 및 서비스 제공
- **일과**: 오전 현장 출동/상담 → 오후 기록 업무 → 퇴근 후 밀린 DB 입력
- **고통**: "상담 끝나고 PC 앞에 앉으면 또 시작이에요. 엑셀에 쓴 거 똑같이 시스템에 치기만 하면 되는데, 드롭다운 하나하나 누르고 복붙하고... 3명 거 하면 한 시간은 넘어요."
- **기술 수준**: 엑셀 중급, IT 도구 사용에 거부감 없음, 프로그래밍 경험 없음

#### 페르소나 B: 박영호 팀장 (42세, 경력 15년)
- **소속**: 전북 아동보호전문기관 사례관리팀장
- **역할**: 팀원 관리, 복지부 보고 데이터 취합
- **고통**: "팀원들이 시스템 입력을 밀려서 월말에 몰아서 하니까 통계가 지저분해요."
- **니즈**: 팀원들의 DB 입력 지연 해소, 실시간에 가까운 데이터 반영

#### 페르소나 C: 이정은 기관장 (50세)
- **역할**: 예산 집행 및 운영 총괄, 위탁 법인 보고
- **의사결정 기준**: 비용 대비 효과(ROI), 규정 준수(컴플라이언스), 직원 복지
- **우려**: "개인정보 유출 문제는 없나요? 보안 감사에 걸리면 안 되는데..."
- **Dash의 답**: 완전 로컬 구동, 서버 없음, 데이터 외부 전송 없음

### 3.2 사용 상황 (Context of Use)
| 상황 | 빈도 | 세부 |
|---|---|---|
| 상담 후 DB 일괄 입력 | 매일 오후 | 당일 진행한 3~10건의 서비스를 퇴근 전 몰아서 입력 |
| 월말 일괄 정리 | 월 1회 | 밀렸던 건들을 한꺼번에 소화 (20~50건) |

| **DOM 이벤트 자동화** | 단순 값 입력을 넘어 change, input 이벤트를 발생시켜 시스템 상태 업데이트 보장 | ✅ 완료 |

| **탭 전환 UI** | 새 파일 업로드 / 수동 입력 / 최근 기록 3개 탭으로 워크플로우 분리 | ✅ 완료 |

| **하이라이트 오버레이** | 자동 입력 시 타겟 페이지에 시각 피드백(초록색 테두리) 표시 | ✅ 완료 |

### 6.2 코드 아키텍처 (3-Layer)

```
dbauto_extension/
├── core/                    ← 핵심 설정 & 상수
│   ├── config.js            ← 필드 ID 매핑, 색상, 타이밍 설정
│   └── message-types.js     ← 모듈 간 통신 액션 상수
├── adapters/                ← 브라우저 API 추상화
│   ├── chrome-messenger.js  ← chrome.tabs.sendMessage 래퍼
│   └── storage-adapter.js   ← chrome.storage.local 래퍼
├── ui/                      ← 사이드 패널 UI 모듈
│   ├── app.js               ← 진입점 (초기화, 공유 상태)
│   ├── tab-manager.js       ← 탭 전환 (업로드/수동/기록)
│   ├── excel-handler.js     ← 엑셀 파일 파싱 (SheetJS)
│   ├── manual-form.js       ← 수동 입력 폼 생성/관리
│   ├── window-manager.js    ← 타겟 창 감지/선택/일괄입력
│   ├── hover-select.js      ← 마우스 호버 기반 레코드 선택
│   └── history-manager.js   ← 입력 기록 관리
├── content.js               ← 타겟 페이지 DOM 제어 (자동 기입)
├── background.js            ← Service Worker (사이드 패널 제어)
├── sidepanel.html/css       ← 사이드 패널 레이아웃
├── popup.html/js            ← 팝업 UI (보조)
├── manifest.json            ← 확장 프로그램 설정
├── privacy-policy.html      ← 개인정보처리방침
└── xlsx.mini.min.js         ← SheetJS 라이브러리 (엑셀 파싱)
```

### 6.3 사용 권한 (최소 권한 원칙)

| 권한 | 용도 |
|---|---|
| `storage` | 폼 상태, UI 설정, 입력 기록을 로컬에 저장 |
| `tabs` | 열려 있는 시스템 창(탭)들의 ID/제목 조회 |
| `sidePanel` | 사이드 패널 UI 제공 |
| `content_scripts` | ncads.go.kr, localhost에서만 DOM 제어 스크립트 자동 주입 |

> ⚠️ `scripting`, `activeTab` 권한은 실제 미사용으로 판단하여 제거 완료 (Chrome Web Store 심사 대응)

### 6.4 핵심 설계 원칙

1. **데이터 비유출 (Zero Data Leakage)**: 아동 개인정보는 절대 외부 서버로 전송하지 않음. 모든 처리는 사용자 브라우저 내에서만 수행
2. **완전 로컬 구동**: 서버·데이터베이스·클라우드 없음. 네트워크 불안정 환경에서도 정상 작동
3. **개인정보 자동 마스킹**: 사이드 패널에서 피해아동 이름을 마스킹(강O정) 처리하여 화면 노출 최소화
4. **최소 침습**: 국가시스템의 API를 호출하는 것이 아니라 UI를 사람처럼 조작하므로, 시스템에 부하를 주지 않음
5. **최소 권한**: Chrome Web Store 정책에 따라 실제 사용하는 권한만 요청

## 8. 유저 케이스 (Use Cases)

### UC-1: 엑셀 기반 다중 창 일괄 입력 (Happy Path)
```
사전 조건: 상담원이 엑셀 파일에 당일 서비스 3건을 기입 완료,
           시스템에서 3명의 아동 팝업창을 미리 열어둠

1. 상담원이 Chrome 확장 프로그램 아이콘을 클릭하여 사이드 패널을 연다
2. '새 파일 업로드' 탭에 엑셀 파일을 드래그 앤 드롭한다
3. Dash가 엑셀을 파싱하여 3건의 레코드를 카드 형태로 보여준다
4. 하단 '입력할 대상 창' 영역에 3개의 시스템 팝업이 자동 감지되어 표시된다
   (각 창 옆에 마스킹된 피해아동 이름 표시: 강O정, 김O수, 이O민)
5. 상담원이 체크박스로 대상 창 3개를 선택한다
6. [자동 입력 시작] 버튼을 클릭한다
7. Dash가 3개 창에 동시에 데이터를 자동 기입한다
8. 입력 완료 후 각 창에 초록색 테두리로 성공 표시
9. 입력 기록이 '최근 기록' 탭에 저장된다

사후 조건: 3건의 서비스 데이터가 각 팝업에 정확히 기입됨
           (저장 버튼은 상담원이 직접 확인 후 클릭)
```

### UC-2: 수동 입력 모드
```
사전 조건: 엑셀 파일 없이 직접 입력하고 싶은 경우

1. 사이드 패널에서 '수동 입력' 탭을 선택한다
2. 감지된 창별로 폼이 자동 생성된다
3. 각 폼에서 드롭다운 선택, 텍스트 입력을 직접 한다
4. 긴 텍스트는 🔍 버튼으로 모달에서 확대 편집한다
5. [입력] 버튼으로 해당 창에 전송하거나, [일괄 입력]으로 전체 전송한다

사후 조건: 수동 입력 데이터가 타겟 창에 기입됨
```

## 9. 유저 저니맵 (User Journey Map)

```

```
        Chrome Web Store에서 'Dash' 검색
              ↓
        [Chrome에 추가] 클릭 → 설치 완료 (5초)
              ↓

              ↓
        "이게 끝이야? 진짜 간단하다!"
```

```
        시스템에서 아동 3명의 팝업창을 열어둠

### Phase 4: 일상 사용 → 확산

        매일 오후 4시, 상담 기록 정리 후 Dash 사이드 패널 열기

              ↓

              ↓

## 13. 수익 모델

### 현재 전략: 무료 배포 (팬덤 확보기)
- Chrome Web Store에서 무료로 배포
- 사용자 기반 확보 및 현장 피드백 수집에 집중
- 서버 비용이 $0이므로 유지 비용 최소

### 향후 전환 옵션

#### Option A: 프리미엄 모델 (Freemium)
- **무료**: 기본 입력 자동화 (현재 기능)
- **유료**: 템플릿 관리, 고급 리포트, 우선 지원 (월 5~10만 원/기관)

#### Option B: 위탁법인 본부 일괄 계약 (B2B)
- 굿네이버스, 초록우산 등 본부와 계약 → 산하 전체 기관에 배포
- 기관당 단가 인하 but 볼륨 확보

> 💡 **추천**: 현재는 무료 배포 + 입소문으로 팬덤 확보 → 충분한 사용자 확보 후 Option A 또는 B로 전환
